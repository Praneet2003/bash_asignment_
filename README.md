🚀 This project brings automation to life by continuously monitoring system services on a Linux server and taking instant action when things go wrong. Instead of manually checking and restarting crashed services, this Bash script intelligently detects failures, attempts recovery, and logs every event with clear timestamps and severity levels.

With built-in fault tolerance, structured logging, and a safe `--dry-run` mode for testing, the script simulates a real-world DevOps monitoring tool. It not only ensures higher service availability but also provides a clear summary of system health, making debugging and maintenance faster and more efficient.

💡 In short: it acts like a lightweight watchdog for your services—always alert, always ready to recover.
